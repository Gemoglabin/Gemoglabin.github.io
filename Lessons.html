Адаптивний дизайн - це розробка веб-сайтів, оптимізованих для перегляду на екранах різних розмірів, таких як настільні комп'ютери, ноутбуки, планшети та смартфони. Метою адаптивного дизайну є створення безшовного користувацького досвіду на всіх пристроях, незалежно від розміру екрана. У цьому посібнику ми розглянемо основні принципи адаптивного дизайну і наведемо кілька прикладів коду для початку роботи.

Крок 1: Використовуйте чуйний дизайн

Першим кроком до створення адаптивного веб-сайту є використання чуйного дизайну. Адаптивний дизайн - це метод проектування веб-сайтів, який використовує медіа-запити CSS для налаштування макета і стилю веб-сайту залежно від розміру екрана використовуваного пристрою. Завдяки чуйному дизайну ваш сайт автоматично адаптуватиметься до різних розмірів екрана, полегшуючи користувачам навігацію та взаємодію з вашим контентом. 

body {
    /* Desktop styles */
}

/* Use media queries to adjust styles for different screen sizes */
@media screen and (max-width: 768px) {
    body {
        /* Tablet styles */
    }
}

@media screen and (max-width: 480px) {
    body {
        /* Mobile styles */
    }

Крок 2: Використовуйте гнучкі макети

Другий крок до створення адаптивного сайту - використання плавних макетів. Рідкі макети - це макети, які підлаштовуються під ширину екрана, а не використовують фіксовану ширину. При використанні плавних макетів ваш сайт буде розширюватися або стискатися, заповнюючи доступний простір на екрані, що полегшує користувачам перегляд вашого контенту незалежно від розміру екрана. 

/* Set the container width to 100% */
.container {
    width: 100%;
}

/* Set the column widths as a percentage */
.col-3 {
    width: 25%;
}

.col-6 {
    width: 50%;
}

У цьому прикладі ми використовуємо поточний макет для створення двоколонкового макета. Ширина контейнера встановлена на 100%, що означає, що він буде розширюватися або стискатися, щоб заповнити доступний простір на екрані. Ширина колонок задана у відсотках, що означає, що вони підлаштовуватимуться під ширину екрана. Колонки розташовуються зліва і справа відповідно, щоб створити двоколонковий макет.

Крок 3: Використовуйте гнучкі зображення

Третій крок до створення адаптивного сайту - використання гнучких зображень. Гнучкі зображення - це зображення, які підлаштовуються під ширину екрана, а не використовують фіксовану ширину. За допомогою гнучких зображень ваші зображення будуть розширюватися або стискатися, заповнюючи доступний простір на екрані, що полегшує користувачам перегляд ваших зображень незалежно від розміру екрана... 

/* Set the max-width of the image to 100% */
img {
    max-width: 100%;
    height: auto;
}

У цьому прикладі ми використовуємо CSS, щоб встановити максимальну ширину зображення на 100%. Це означає, що зображення буде підлаштовуватися під ширину екрана, але не перевищуватиме свій початковий розмір. Ми також встановлюємо висоту на auto, що забезпечує збереження співвідношення сторін зображення під час його адаптації до різних розмірів екрана..

Крок 4: Адаптивна типографіка

У цьому уроці ми розглянемо, як створити адаптивну типографіку на вашому сайті. Адаптивна типографіка дає змогу тексту адекватно відображатися на різних пристроях і розмірах екранів.

Замість використання фіксованих значень шрифтів, ми будемо використовувати відносні одиниці виміру, такі як vw (viewport width) або rem (відносний розмір шрифту). Це дасть змогу тексту пропорційно масштабуватися зі зміною розміру екрана.

Встановіть базовий розмір шрифту для кореневого елемента (html або body). Цей розмір буде використовуватися в якості відправної точки для інших елементів. 

html {
  font-size: 16px; /* Початковий розмір шрифту */
}

Тепер ми можемо застосувати адаптивну типографіку до різних елементів вашого сайту. Наприклад, припустимо, ви хочете застосувати адаптивний розмір шрифту до абзаців на вашій сторінці. 

.adaptive-text {
  font-size: 4vw; /* Розмір шрифту залежно від ширини екрана */
}

У цьому прикладі ми використовували одиницю виміру vw (viewport width), яка дає змогу шрифту масштабуватися пропорційно ширині вікна перегляду (viewport). Таким чином, розмір шрифту буде автоматично змінюватися при зміні ширини екрана.

Для створення більш точного контролю над адаптивною типографікою ви можете використовувати медіа-запити. Медіа-запити дають змогу змінювати розміри шрифтів на певних точках зупинки (breakpoints) залежно від різних розмірів екранів. 

@media (max-width: 767px) {
  .adaptive-text {
    font-size: 2rem; /* Новий розмір шрифту для мобільних пристроїв */
  }
}

У цьому прикладі ми застосовуємо новий розмір шрифту, використовуючи відносну одиницю виміру rem (відносний розмір шрифту), коли ширина екрана не перевищує 767 пікселів. Це дасть змогу тексту на мобільних пристроях бути більш читабельним і зручним для користувача.

Шляхом комбінування відносних одиниць виміру, базового розміру шрифту та медіа-запитів ви зможете створити адаптивну типографіку, що матиме чудовий вигляд на будь-якому пристрої та розмірі екрана.

Крок 5: Використовуйте дизайн, орієнтований на мобільні пристрої

Останнім кроком у створенні адаптивного веб-сайту є використання мобільно-первісного дизайну. Mobile-first design - це підхід до проектування, за якого пріоритет віддається мобільним пристроям, а не настільним комп'ютерам. За мобільно-первісного дизайну ви починаєте з проєктування для найменшого розміру екрана, а потім переходите до більших розмірів екрана. 

/* Default styles for all screen sizes */
body {
    font-size: 1rem;
}

/* Styles for screens 768px and larger */
@media screen and (min-width: 768px) {
    body {
        font-size: 1.5rem;
    }
}

Висновок

Створення адаптивного веб-сайту вимагає поєднання чуйного дизайну, плавних макетів, гнучких зображень і дизайну, орієнтованого на мобільні пристрої. Дотримуючись цих принципів і використовуючи наведені приклади коду, ви зможете створити сайт, який матиме чудовий вигляд на всіх пристроях, незалежно від розміру екрана. Ви можете подивитися, який вигляд матиме той код, що ми використовували, на великому і маленькому екрані. Також наприкінці додається файл із вихідним кодом. Створений приклад
