Медіазапити

Раніше створення веб-сайту було набагато простішим. Сьогодні макет веб-сайту повинен підлаштовуватися не тільки під комп'ютери, а й під планшети, мобільні пристрої і навіть телевізори.

Створення веб-сайту з адаптованим макетом називається адаптивним веб-дизайном. І CSS Media Queries - одна з найважливіших частин адаптивного дизайну. У цій статті ми детальніше розглянемо медіазапити та способи їх використання в CSS.

Що таке медіазапит?

Медіа-запит - це функція CSS3, яка змушує веб-сторінку адаптувати свій макет до різних розмірів екрана і типів мультимедіа.

Синтаксис 

@media media type and (condition: breakpoint) {
  // CSS rules
}

Ми можемо орієнтуватися на різні типи медіа в різних умовах. Якщо умова та/або типи мультимедіа відповідають, то будуть застосовані правила всередині медіа-запиту, в іншому разі вони не застосовуватимуться.

Спочатку синтаксис може здатися складним, тому давайте детально пояснимо кожну частину...

@ Медiа Правило

Ми починаємо визначати медіазапити з правила @media, а пізніше включаємо правила CSS всередині фігурних дужок. Правило @ media також використовується для зазначення цільових типів мультимедіа. 

@media () {
  // CSS rules
}

Усередині правила

Усередині дужок ми встановлюємо умову. Наприклад, я хочу застосувати більший розмір шрифту для мобільних пристроїв. Для цього нам потрібно встановити максимальну ширину, яка перевіряє ширину пристрою: 

.text {
  font-size: 14px;
}

@media (max-width: 480px) {
  .text {
    font-size: 16px;
  }
}

Зазвичай розмір тексту буде 14px. Однак, оскільки ми застосували медіа-запит, він зміниться на 16 пікселів, коли максимальна ширина пристрою становитиме 480 пікселів або менше.

Важливо: завжди розміщуйте медіа-запити в кінці файлу CSS.

Типи носіїв

Якщо ми не застосовуємо тип носія, правило @media за замовчуванням вибирає всі типи пристроїв. В іншому випадку типи Media йдуть відразу після правила @media. Існує багато видів пристроїв, але ми можемо розділити їх на 4 категорії:

  • all - для всіх типів носіїв
  • print — для принтерів
  • screen — для екранів комп'ютерів, планшетів і смартфонів
  • speech — для програм читання з екрана, які "читають" сторінку вголос

Наприклад, коли я хочу вибрати тільки екрани, я встановлю ключове слово screen відразу після правила @media. Я також повинен об'єднати правила за допомогою ключового слова "and": 

@media screen and (max-width: 480px) {
  .text {
    font-size: 16px;
  }
}

Контрольні точки

Контрольна точка, мабуть, найпоширеніший термін, який ви почуєте і будете використовувати. Контрольна точка - це ключ до визначення моменту зміни макета й адаптації нових правил всередині медіа-запитів. Повернемося до нашого прикладу на самому початку: 

@media (max-width: 480px) {
  .text {
    font-size: 16px;
  }
}

Тут точка контрольна точка 480px. Тепер медіа-запит знає, коли встановити або перезаписати новий клас. По суті, якщо ширина пристрою менша за 480 пікселів, буде застосовано текстовий клас, в іншому разі - ні.

Загальні контрольні точки: чи є стандартна роздільна здатність?

Одне з найбільш поширених запитань: "Яку контрольну точку слід використовувати?". На ринку існує безліч пристроїв, тому ми не можемо і не повинні визначати фіксовані точки зупинки для кожного з них.

Ось чому ми не можемо сказати, що є стандартний дозвіл для пристроїв, але є деякі часто використовувані контрольні точки в повсякденному програмуванні. Якщо ви використовуєте фреймворк CSS (наприклад, Bootstrap, Bulma тощо), ви також можете використовувати їхні контрольні точки.

Тепер давайте подивимося на деякі загальні контрольні точки для ширини пристроїв:

  • 320 пікселів - 480 пікселів: мобільні пристрої
  • 481px - 768px: iPad, планшети
  • 769px - 1024px: маленькі екрани, ноутбуки
  • 1025px - 1200px: настільні комп'ютери, великі екрани
  • 1201px і більше - дуже великі екрани, телевізор

Як я сказав вище, ці контрольні точки можуть відрізнятися, і немає точно визначеного стандарту, але це деякі з найбільш часто використовуваних.
